# Milestone v2.1: Engagement Features

**Status:** SHIPPED 2026-05-09
**Phases:** 10-11
**Total Plans:** 2

## Overview

用户参与度提升：成就徽章激励 + 扫码快速发布流程。

## Phases

### Phase 10: Achievement System

**Goal**: Users earn points and badges for eco-friendly actions
**Requirements:** ACH-01 ~ ACH-06

- eco_points already in users table + awarded on completion (listing-complete.mjs)
- badges.js: 6 badge definitions (FIRST_PUBLISH, FIRST_COMPLETE, CARBON_10/50/100, TRADE_10)
- user_badges table in Turso DB
- checkAndAwardBadges() called from listing-create + listing-complete
- user-profile.mjs returns earned badges + all definitions
- me.js: identity card shows eco_points + badge count, badge shelf grid
- publish.js / listing-detail.js / me-listings.js: badge toast on earn

### Phase 11: QR Scan Quick Publish

**Goal**: Camera capture → AI recognition → 1-tap publish
**Requirements:** QR-01 ~ QR-06

- scan.js: new page with getUserMedia camera viewfinder
- Photo capture, compressForApi, analyzeWithAI in one flow
- Simplified pre-filled form → one-tap submit
- File picker fallback when camera denied
- Front/back camera switch
- /scan route (auth-protected)
- publish.js: "拍照快速发布" entry point
- sw.js: scan.js precached

---

## Milestone Summary

**Key Decisions:**
- Badge definitions hardcoded in badges.js (not DB-driven) — simpler, no admin UI needed
- getUserMedia with facingMode: environment as default — back camera for item scanning
- compressForApi reuse — same 800px/0.7 compression as regular upload

**Issues Resolved:**
- eco_points already existed in schema — no migration needed for ACH-01
- API completion flow already existed in api-adapter.js — just needed badge response handling
