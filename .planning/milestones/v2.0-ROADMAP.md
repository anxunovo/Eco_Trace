### Phase 8: Data & Impact Visualization

**Goal**: Users see their carbon impact through interactive charts and real-time statistics
**Depends on**: v1 complete (Phase 1-4)
**Plans**: 3 plans

Plans:
- [x] 08-01: Backend — Extend carbon-stats API (period filter, trend, campus comparison)
- [x] 08-02: Frontend — Chart.js + charts.js module (TrendChart, CategoryDoughnut, CampusBar)
- [x] 08-03: Frontend — Impact page + Homepage animated counters

**Details:**

- Chart.js 4.x loaded via CDN (no build step)

- Impact page: trend line chart, category doughnut, campus bar chart, period filter tabs
- Homepage: AnimatedNumber component (ease-out cubic rAF) on all stat cards
- UAT: 5/5 criteria PASS, 0 issues

### Phase 9: PWA Foundation

**Goal**: App works offline and can be installed on mobile devices
**Depends on**: Phase 8
**Plans**: 3 plans

Plans:

## Milestone Summary

**Key Decisions:**
- Chart.js via CDN (no build step) — keeps zero-build architecture
- Service Worker v3 — POST/PUT/DELETE passthrough, only GET cached
- glm-4v-flash model (not glm-4.6v-flash) — verified stable, no rate limits

**Issues Resolved:**
- BUG-003: API probe permanently cached → 30s TTL
- BUG-004: SW caches POST requests → passthrough
- BUG-005: image-utils.js dead code → compressForApi() active
- BUG-006: Model name wrong → verified glm-4v-flash correct
- BUG-007: Multi-image ignored → parallel per-image requests + merge

**Issues Deferred:**
- iOS apple-touch-icon SVG limitation (cosmetic)
- Chart.js CDN not cached on first offline visit (edge case)

**Technical Debt:**
- No frontend build system (deferred to v2.1+)
- localStorage still used as offline fallback (not just API cache)
